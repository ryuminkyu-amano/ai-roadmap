흩어진 엑셀 파일에서 출발해, 팀 공유 DB를 거쳐 사내 MCP 시스템까지— 단계적으로 구축하는 4단계 전략입니다.

PHASE 01
Cowork 파일럿
개발 없이 지금 바로 · 가능성 검증

로컬 폴더를 지정하고 엑셀 파일 몇 개를 넣어둔 뒤, Claude Cowork에 자연어로 리포트·조건검색을 요청합니다. 코드 한 줄 없이 즉시 시작할 수 있고, AI 분석의 가능성과 한계를 빠르게 체감할 수 있습니다.

PHASE 02
자체 DB 구축 (개인 / 소팀)
엑셀 → DB 마이그레이션 · Cowork가 DB에 직접 질의

기존 엑셀들을 ETL 스크립트로 정제해 로컬 DB에 적재합니다. Cowork가 DB에 직접 접근하거나, 자연어 → SQL 변환 방식으로 정확한 조건검색과 집계 리포트를 처리합니다. Phase 1의 한계(매번 파일 재읽기, 느린 검색)를 근본적으로 해결합니다.

PHASE 03
팀 공유 DB (사내 인프라)
중앙 DB 서버 · 팀간 공통 접근 · 권한 관리

RDS / PostgreSQL 같은 중앙 DB 서버로 전환하고, 팀별 롤과 스키마를 분리해 접근 권한을 설계합니다. 팀원 각자가 Cowork를 통해 동일한 DB에 AI 분석을 요청할 수 있는 구조로, 데이터 사일로 없이 조직 전체가 같은 맥락으로 일할 수 있습니다.

PHASE 04
사내 MCP 시스템 구축
Claude API + 자체 MCP 서버 · 전사 AI 인프라

Claude API 기반의 자체 MCP 서버를 개발하고, DB·내부 API를 툴로 등록합니다. 전사 Claude 연동으로 Cowork를 넘어 사내 시스템 전반—업무 자동화, 실시간 운영 리포트, 승인 워크플로우 등—을 AI가 직접 처리하는 인프라가 완성됩니다.
